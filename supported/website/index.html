This is using elib1_new_webkit batch server. Basically there are two forms of URL:

  1. Regular links, like: <a href="foo.html">foo</a>. Clicking on a regular link just fetches the item concerned.
  2. Module links, like: <a href="mod?mod=cedit&func=edit&file=/examples/ebook/estore1.xml">Edit George</a> Module links evaluate code. In the example the code is in the mode cedit.erl - cedit will be dynamically recompiled if it is outofdate

At the moment the code for the website is spread out all over the place. I'll merge it later when stuff works. The general idea is not to break things by moving too much at any one time.

The website has an on-line and off-line component. If you have downloaded the site and are running locally you can do a lot more than if you are are remotely viewing the site. Off line you can only read the documentation, but if you download the stuff you can interact with the programs.

This is a library of erlang modules

When you have installed the library you should also have a function web server that you can use to interact with the applications.

There are also a number of applications that are written using the modules in the library.

To view this web site and to interact with the applications it is assumed that you are running the Erlang web server which is an integral part of the library.

Top start the system and view the website you should give the command "make" in the top-level directory of the website.
